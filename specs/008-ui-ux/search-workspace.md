# 008 搜索工作区 (Search Workspace)

定义 FauniSearch 搜索工作区的共享界面与体验约束，明确不同查询能力专题如何在同一搜索工作区中进入、展示结果并展开详情。

## 关键术语 (Terminology)

- 搜索工作区（Search Workspace）
- 查询模式（Query Mode）
- 统一搜索入口（Unified Search Entry）
- 结果列表（Result List）
- 详情面板（Detail Panel）
- 预览资源引用（Preview Resource Reference）

## 范围

- 搜索工作区内部的共享布局与交互骨架
- 查询模式切换与统一搜索入口的共享约束
- 结果列表、详情面板、预览与共享状态反馈的工作区语义
- 搜索工作区与当前库上下文、嵌入式任务观察面的关系

范围外：
- 具体查询能力的输入语义、结果对象边界与阶段范围
- `/search/*`、详情接口或上传接口的请求 / 响应编码
- 搜索排序、过滤、分页、调试载荷与错误码触发条件
- 应用壳层、正式工作区列表与全局导航关系
- 像素级布局、视觉样式与具体组件实现

## 设计原则

- 工作区连续性（Workspace Continuity）：搜索输入、结果浏览与详情浏览应在同一工作区内连续完成，而不是频繁跳转独立页面
- 当前库上下文显式（Visible Library Context）：搜索工作区必须始终让用户知道当前查询作用于哪个库
- 统一入口、单次单输入（Unified Entry, Single Query Input）：工作区可以承接统一搜索入口与查询模式切换，但单次公开搜索请求只应发出一种查询输入
- 最小结果优先（Minimal Result First）：结果列表默认呈现最小结果卡片；更完整的信息通过详情面板继续展开
- 共享状态明确（Explicit Shared States）：空结果、未就绪与失败在工作区中必须可区分，不得互相伪装

## 工作区骨架

- 搜索工作区必须与当前库上下文共同呈现，用户进入工作区后无需离开当前页面即可完成查询、结果浏览与详情查看
- 搜索工作区可以采用多栏或等价分区布局，但必须同时满足以下共享关系：
  - 查询输入区与结果列表处于同一连续工作流中
  - 结果详情在同一工作区内展开，而不是强制跳转独立详情页
  - 嵌入式任务观察面可以与搜索工作区共存，用于表达导入、索引或相关后台动作的进度与失败摘要
- 搜索工作区可以内嵌导入或查询前置动作入口，但这些入口不应改写搜索能力专题本身的输入语义

## 查询入口与模式切换

- 搜索工作区可以提供统一搜索入口，用于承接不同查询能力专题的查询模式
- 查询模式切换属于工作区体验的一部分，但具体有哪些正式查询模式由各搜索能力专题定义
- 统一搜索入口下，工作区必须避免把多种查询输入伪装成同一次公开搜索请求
- 工作区可以记住当前会话中的查询模式选择，但该状态不改变公开搜索端点的单次单输入约束

## 结果与详情关系

- 结果列表默认返回最小结果卡片，并保持稳定的列表浏览体验
- 点击或选中结果后，工作区必须在同一页面内打开详情，而不是要求用户跳转独立对象页
- 详情面板同时承接可视预览与结构化信息
- 可视预览优先通过 app 提供的预览资源引用展示，而不是继续依赖裸本地路径或 `file://`
- 结果列表与详情面板的具体字段由各搜索能力专题复用 [004-search](../004-search/spec.md) 与 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md) 的事实源

## 共享状态反馈

- 当查询尚无结果时，工作区必须表达明确空结果状态
- 当目标索引线尚未 ready 时，工作区必须表达明确未就绪状态，而不是只显示空列表
- 当查询、导入或索引相关动作失败，并影响用户在工作区中的下一步时，工作区必须表达明确失败反馈
- 这些共享状态的触发语义与错误载荷编码继续分别复用 [004-search](../004-search/spec.md)、[006-runtime-and-execution](../006-runtime-and-execution/spec.md) 与 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md)
