# 008 当前阶段 UI 落地目标 (Current UI Targets)

记录当前阶段 UI 重构的落地目标与实现建议。本文件服务当前实施，不定义长期不可变规范；若与 `008` 其它正式子文档冲突，以正式事实源为准。

## 文档角色

- 本文件承接不宜直接写成长期刚性规范的宽度、比例、组件表达与响应式目标
- 本文件用于帮助当前阶段 UI 实施对齐同一方向，而不是冻结未来所有视觉与布局决定

## 当前阶段整体目标

- 把当前界面从“开发 / 运维工作台”收口成“面向用户的产品工作台”
- 优先服务首次用户，但不牺牲长期重度用户的效率
- 当前实施优先级固定为：
  - Search 首页
  - Settings
  - Inventory
  - 统一壳层与状态组件

## 当前补充原型探索目标

- 当前生产落地当前应承接补充原型中已经稳定的软件式壳层，而不是继续沿用“顶部当前库 cluster + 顶部工作区切换”的网页式组织
- 当前生产落地中的顶部只保留品牌与全局状态，不再重复承载当前库范围语义
- 当前生产落地中的左侧栏当前顺序固定为：
  - `Search`
  - `库管理`
  - `工具`
  - `设置`
- 当前生产落地中，`库管理` 承接 `Inventory` 的可见入口，并吸收来源展示与库级管理动作
- 当前生产落地中的搜索框当前目标采用：
  - 左侧主输入
  - 右侧 `过滤` 按钮
  - 右侧 `图片 / 视频 / 文档` icon-only 入口
- 当前生产落地中的库上下文当前目标改为 composer 下方的范围条；当前第一版正式支持 `当前库` 与 `所有库`，并为未来 `多库` 预留样式与交互扩展位
- 当前补充原型中的最近查询当前目标需要同时支持文本、图片、视频与文档查询历史的可视表达，而不是只接受纯文本字符串；生产实现若当前没有正式历史数据，可暂不伪造该区域
- 当前生产落地当前目标可以直接采用已经稳定的极简英文表达：品牌副标题 `Unified · Native · Powerful`、顶栏状态词 `Ready`、空态搜索标题 `Search anything you want`

## 当前阶段壳层目标

- 桌面端主版心当前目标约为 `1440px`
- 上下文条当前目标收敛为单层紧凑 app bar，不再承担 hero 式说明块
- 上下文条当前目标优先呈现：品牌与极短 tagline、状态胶囊；不再保留“本地多模态搜索 / 统一检索...”式标题与说明块
- 上下文条当前目标应继续压缩按钮、标签与密度，避免重新长成“标题区 + 库卡 + 动作区”的网页式三段结构
- `Search` / `Inventory` / `Settings` 在同一壳层内切换
- 状态入口与工具入口可以共同打开同一个右侧 utility drawer
- 桌面端当前目标只允许一个辅助面存在；不得同时出现状态卡片和工具卡片两个大型浮层

## 当前阶段 Search 目标

- 搜索首页当前目标骨架为：
  - 顶部轻量 `app bar`
  - 左侧稳定导航
  - 右侧主内容中的整宽 `Search stage`
  - 有结果时的下层 `Results / Detail` 阅读面
- `Search stage` 当前目标结构为四层：
  - 极简主任务标题
  - active composer（含 `过滤` 按钮与非文本模式 icon）
  - 搜索范围条
  - CTA + 过滤摘要 / 展开区
  - state / help strip
- 空态搜索舞台头部当前目标不再保留 `搜索` eyebrow、`开始搜索` 中文标题与右侧解释文案；只保留单句主任务表达，例如 `Search anything you want`
- 空态搜索舞台头部当前目标应居中呈现，不再像普通表单标题那样左对齐
- 搜索范围条当前目标除范围切换外，只保留当前库选择；`新建库 / 管理当前库` 全部移到 `库管理` 工作区
- 搜索范围条当前目标不再只是摘要文字，而应成为正式范围选择控件；当前至少支持：
  - `当前库`
  - `所有库`
- 当前阶段中，当范围为 `所有库` 时：
  - `Text` 继续可用
  - `Image` / `Video` / `Document` 当前应显式回退到 `当前库` 或给出明确不可用提示
- 当前阶段中，当范围为 `所有库` 时，`submit`、状态条、next-step 与空结果反馈都应跟随“当前范围里是否已有可搜索库”，而不是继续按当前选中库单独禁用或报未就绪
- 搜索舞台顶部当前只保留最小标题与一行帮助语，不再保留 marketing 风格说明块
- 桌面端有结果后，`Search stage` 当前目标仍然作为整宽首行保留在上方；不得重新退化成左栏 `Search + Results + Detail` 三栏并排
- 桌面端有结果后，`Search stage` 当前目标还应切到更薄的 reading 变体：更紧的头部、更低的 composer、高度受控的预览面，以及更像工具条的 CTA / filters 区
- `Search stage` 当前目标采用：
  - 左侧主 composer
  - 右侧 `next-step dock`
  - `dock` 仅在需要时出现
- 常驻过滤器当前只保留：
  - 视觉对象类型
  - 来源类型
- `Advanced` 当前目标为行内展开，不做抽屉
- 导入入口当前目标只放在 `next-step dock`
- `Image` / `Video` / `Document` 当前目标继续共用同一种查询对象卡语法：
  - 顶部输入来源与状态
  - 中部预览面
  - 下部范围 / 专题控件与清除动作
- 搜索成功后当前目标是自动选中第一条结果
- 无结果、无库、无内容与 ready 空态下，桌面端当前目标不再保留独立结果列或详情列占位
- 结果卡当前目标为：
  - 左侧小缩略图（当前优先压到约 `64-72px` 级别）
  - 右侧信息
  - `score` 中等可见
  - 桌面端 hover 显示复用动作
  - 触屏端选中后显示复用动作
- 当 `所有库` 范围返回跨库结果时，结果区当前目标还应在列表之上补一层命中库分布摘要，详情面当前目标应提供进入命中库库管理上下文的直接动作，而不是只在结果卡里堆更多标签
- 当 `所有库` 范围返回跨库结果时，命中库分布当前目标应直接支持“全部结果 / 某个命中库”的聚焦切换；如果当前详情对象不在新的聚焦组里，详情面应自动切到该组第一条结果
- 当 `所有库` 范围返回多个命中库且当前仍停留在“全部结果”时，结果列表当前目标应以命中库为单位形成分组标题，让跨库阅读先有秩序，再决定是否聚焦到单一库
- 这些分组标题当前目标不只是库名行；还应补上每组结果数量与快速动作，至少包括：
  - `仅看这个库`
  - `在库管理查看`
- 当跨库阅读已经由分组标题或焦点摘要承担库身份时，结果卡当前目标不再重复显示库 pill，把信息密度留给对象类型、页码 / 时间片、相似度与路径
- 结果卡当前目标中的对象信息顺序应固定为：
  - `对象类型 / 页码或时间片 / 相似度`
  - `文件名`
  - `路径`
- grouped 跨库阅读态下，结果卡当前目标应更紧凑；focused 或单库阅读态下，可以给文件名更强一点的标题权重，但仍不恢复库身份重复表达
- grouped 跨库阅读态下，结果工作面当前目标应整体更轻，分组标题比结果卡更显眼；focused 或单库阅读态下，结果卡当前目标可以恢复更强对比和更清晰的对象阅读权重
- 当 `所有库` 范围下从结果或详情复用对象为下一次查询输入时，当前目标应自动切换到命中库并回到 `当前库` 范围，让库绑定查询模式保持真实语义
- 结果卡当前应进一步压缩装饰性背景、圆角和阴影，让扫读效率优先于展示感
- 详情面当前目标为独立 sticky 阅读面，首屏预览优先，主动作位于顶部动作带
- `next-step dock` 当前目标应更像轻量邻接提示，不再以强背景、强层级与主查询输入卡平起平坐
- `Search stage` 与 `next-step dock` 当前不再保留 `fill demo`、`run demo`、`快速验证` 等内部验证快捷入口
- `Search stage` 当前目标应复用 `库管理` 当前库管理带的 readiness 语义；状态条与 `next-step dock` 当前至少要区分：
  - 尚未接入来源根 / 来源根已停用
  - 来源根已接入但还没有可搜索对象
  - 当前库覆盖或 resolved model 仍未就绪
  - 后台任务仍在准备当前库
- 当前库已经可搜索时，状态条当前目标应默认收起，不再重复展示“当前已可搜索”说明
- 搜索执行中的当前目标反馈应精简为结果区顶部单行 `搜索中...`，不再在搜索舞台内部重复铺开长句状态提示
- 文本搜索 composer 当前目标应压缩为单行输入带，搜索图标与输入框同层；正式 `Search` 提交按钮当前目标应进入范围条右侧，并与库选择保持同一行
- 搜索范围条当前目标中的 `当前库 / 所有库`、库选择器与正式 `Search` 按钮应留在同一水平主行；过滤摘要若存在，应退到次行
- 结果工作面当前目标应继续向原型收口：去掉 `搜索 / 结果` 与 `详情 / 结果详情` 这类冗余栏位标题，只保留必要的命中摘要与当前对象信息
- `document_page` 结果卡当前目标不再加载实时 PDF 缩略图；列表中只保留轻量文档占位，避免长 PDF 反复闪黑
- 轻量文档占位当前目标只表达文档类型 `PDF`，不再在缩略图区重复显示“文档页”或页码
- 详情面当前目标不再重复结果卡已经提供的 `打开预览 / 作为查询输入` 动作；这些动作统一留在结果条目侧
- 结果缩略图区当前目标不再保留任何额外的纵向对象类型字样；如果占位已经写明 `PDF`，缩略图区其余类型提示全部移除
- 详情标题区当前目标不再重复输出完整来源路径；路径只在下方结构化信息中保留一处
- `Search stage` 当前目标中的下一步动作不再停留在泛化文案；来源问题优先指向 `库管理` 或搜索舞台中的来源准备，配置问题优先直达 `设置 > 当前库覆盖`
- Search 的空结果与失败 notice 当前目标也应与这套 readiness 词汇一致：
  - 空结果强调“当前库可搜索，但本次查询未命中”
  - 配置相关失败可直接跳去 `设置 > 当前库覆盖`
  - utility drawer 的状态摘要也应给出同一套下一步，而不是只展示系统体检
- 尚未发起查询时，当前目标不再保留额外的结果占位解释卡

## 当前阶段首用与状态目标

### 无库

- `Search stage` 仍然存在
- composer 明显受限，但不完全消失
- 右侧主引导卡主推 `新建库`
- 若系统已有库但未选中，则主推 `选择库`，次给 `新建库`

### 有库但无内容

- composer 继续受限，但弱于“无库”状态
- 右侧主引导卡当前需要继续区分：
  - 尚未接入来源根时主推 `接入第一个来源根`
  - 来源根已接入时主推 `准备第一批内容`
  - 配置未就绪时主推 `检查当前库覆盖`
  - 后台任务进行中时主推 `查看任务`

### not_ready

- 主搜索按钮保留但禁用，并明确说明原因
- 搜索舞台内部只提供轻量任务摘要，不展开成长任务中心

## 当前阶段 Settings 目标

- 设置页当前目标采用左侧章节导航 + 右侧活动工作面
- 左侧章节导航当前目标不再只是文字列表，而是带简短职责提示的章节入口；激活态应明显形成“当前正在编辑这一章”的导航记忆点
- 右侧活动工作面当前目标采用：
  - 章节概览头
  - 2 到 4 个与当前章节最相关的摘要卡
  - 再进入真实编辑面 / 对照面 / 诊断面
- 章节顺序当前固定为：
  - 内容类型
  - 当前库覆盖
  - 连接
  - 模型测试
  - 诊断
- `内容类型` 与 `当前库覆盖` 当前是最主要入口
- `内容类型` 与 `当前库覆盖` 当前都采用四张轻卡切换 Text / Image / Video / Document
- `当前库覆盖` 当前以 `继承默认 | 覆盖当前库` 作为第一控制
- `当前生效结果` 当前目标与库级覆盖编辑面同时可见
- `模型测试` 当前目标采用“上配置，下结果”实验台结构

## 当前阶段 Inventory 目标

- 来源浏览当前目标采用：
  - 顶部当前库管理带
  - 顶部工作区标题与一行浏览任务摘要
  - 来源状态摘要卡
  - 顶部过滤条
  - 左侧中密度来源列表
  - 右侧预览优先的来源详情
- 当前库管理带当前目标采用：
  - 左侧当前库身份与生命周期摘要
  - 一条解释当前库“为什么可搜 / 不可搜”的 readiness 摘要
  - 中间 3 到 4 个小型指标卡（例如来源根、来源记录、可搜索对象、待处理任务）
  - 一组轻量来源根健康快照，用于提示监视中 / 需关注 / 已停用
  - 当前可以附带进入 `设置 > 当前库覆盖` 的直接入口，特别是在内容类型绑定或 resolved model 需要关注时
  - 右侧显示名称编辑与归档 / 恢复 / 删除动作
- 顶部当前库管理带当前应让 `库管理` 这个导航名真正成立，而不是继续把库级生命周期动作藏在壳层 popover 中
- 来源浏览当前目标可以在过滤条附近放入次级动作：
  - 前往来源准备
  - 刷新当前库
  - 重扫当前库
- 来源列表当前目标为：
  - 左侧 `64-72px` 级别前导缩略图或等价材质
  - 中间文件名 / 路径 / 来源根与类型信息
  - 右侧状态与可搜索对象数
- 来源详情当前目标保持 sticky 阅读面，预览优先，动作带贴近预览而不是下沉到技术字段之后
- `refresh / rescan` 当前目标保持次级动作，不抢主浏览流程

## 当前阶段组件目标

### tabs

- 当前目标是产品化一级模式 tabs，而不是后台 segmented control
- 选中态以底面、细边框或下缘强调表达

### buttons

- 主按钮当前目标为中等圆角、稳重矩形
- 次按钮当前目标为克制的纸面操作控件
- 少数主动作可带极小图标

### forms

- 输入控件当前目标为平整纸面输入，而不是浏览器默认控件感
- `select` 当前目标为轻度产品化，而非完全重绘式组件
- 错误状态当前目标以轻边框变化 + 下方解释表达
- Search 主输入比普通设置字段大一档
- 文件上传当前目标使用自定义上传卡

### pills / badges

- 当前目标使用低饱和信息标签
- 状态色只用于关键状态

## 当前阶段响应式目标

- 桌面端：结果与详情保持双栏
- 平板端：结果与详情上下堆叠
- 手机端：详情优先使用 bottom sheet 或等价承载方式

## 当前阶段验收目标

- 用户能从首页一眼理解产品主路径
- `搜索 / 来源浏览 / 设置` 切换保持同一产品壳层感
- Search 首页主舞台明确，管理能力后置
- Settings 呈现为章节化设置页，而非 panel 陈列
- Inventory 呈现为来源浏览器，而非后台表
- 主界面术语与文案以中文任务语言为先，技术词后置
