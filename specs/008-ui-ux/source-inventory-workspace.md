# 008 来源浏览工作区 (Source Inventory Workspace)

定义 FauniSearch 来源浏览工作区的共享界面与体验约束，明确库级来源观察如何作为正式管理工作区存在，并与搜索工作区形成清晰分工。

## 关键术语 (Terminology)

- 来源浏览工作区（Source Inventory Workspace）
- 来源过滤条（Inventory Filter Bar）
- 来源列表（Inventory List）
- 来源详情面（Inventory Detail Surface）
- 工作区切换器（Workspace Switcher）

## 范围

- 来源浏览工作区内部的共享布局与交互骨架
- 库级来源过滤、列表浏览、状态摘要与来源详情的共享观察语义
- 来源浏览工作区与搜索工作区之间的切换关系

范围外：
- 来源根生命周期、`refresh` / `rescan` / watcher 的底层语义
- `/libraries/:id/sources` 等接口的请求 / 响应编码
- 来源修复、人工纠错或 source-level detail 的独立工作流
- 设计 token、像素级布局与具体组件实现

## 设计原则

- 管理面独立（Management Surface Is Separate）：来源浏览属于正式管理 / 观察面，不应继续占据搜索入口上方的主流位置
- 当前库显式（Visible Library Scope）：来源浏览工作区必须始终保持当前库上下文可见
- 浏览器而非后台表（Browser, Not Admin Grid）：工作区的核心任务是浏览、核对与理解来源状态，而不是堆满运维表格
- 过滤可持续（Persistent Filtering）：来源根、来源类型与来源状态过滤应在工作区内保持稳定，不因切换搜索模式而被重写
- 搜索草稿不受扰（Search Drafts Stay Intact）：用户在搜索工作区中的查询草稿、结果与详情选中态，不得因为切换到来源浏览工作区而丢失
- 阅读式详情延续（Readable Detail Continuity）：来源详情应与搜索详情共享阅读式产品语法，而不是退化为开发者侧栏

## 工作区骨架

- 来源浏览工作区与搜索工作区共享同一应用壳层、当前库上下文与全局反馈区域
- 来源浏览工作区至少应同时呈现：
  - 当前库上下文
  - 当前库管理带或等价管理卡
  - 工作区标题与一行浏览任务摘要
  - 来源状态摘要或等价概览卡
  - 来源过滤条
  - 中密度来源列表
  - 预览优先的来源详情面
- 来源浏览工作区可以显示来源摘要，但其首要任务仍然是过滤、浏览与核对来源，而不是把摘要条做成新的管理中心
- 来源浏览工作区中的当前库管理带当前可以承接显示名称编辑、归档 / 恢复与删除等当前库生命周期动作，但这些动作必须清楚从属于“当前库”而不是与来源列表混成一张后台控制表
- 来源浏览工作区中的当前库管理带当前还应直接解释“这个库为什么可搜 / 不可搜”，并优先利用来源根状态、watcher 状态、待处理任务与可搜索对象数来表达
- 当当前库管理带发现库级内容类型覆盖或 resolved model 处于需关注状态时，工作区当前可以提供进入 `Settings > 当前库覆盖` 的直接入口
- 来源浏览工作区可以在标题区或过滤区附近承接次级动作，例如前往来源准备、刷新当前库或重扫当前库；这些动作必须弱于浏览主路径

## 过滤、列表与详情

- 来源过滤条至少应支持：按来源根、来源类型与来源状态进行过滤
- 来源过滤条可以采用粘性区块或等价承载方式，但不应遮挡列表内容
- 来源列表应优先呈现中密度、可扫读的信息，而不是高密度数据表
- 来源列表当前可以为每条来源增加代表性缩略图或等价前导材质，帮助用户快速判断“这是一张图、一段视频还是一个文档来源”
- 来源列表至少应覆盖：
  - `source_path`
  - `source_root_label`
  - `source_type`
  - `kind`
  - `status`
  - `visual_unit_count`
  - `status_reason`（若存在）
- 来源列表当前可以把文件名与完整路径拆开表达：文件名用于首行扫读，完整路径与来源根标签退到次行或元信息行
- 来源详情面应优先承接预览与用户可读信息；低频技术字段可以后置
- 来源详情面当前可以先给出当前状态、来源类型、可搜索对象数与代表性对象摘要，再进入更低频的库归属、状态原因或技术背景
- 当前库管理带当前可以附带轻量来源根健康快照，用于说明哪些来源根正在监视、哪些来源根需要关注；这类快照从属于当前库管理，不取代完整来源根编辑面
- 当当前筛选条件下无来源时，工作区必须显示明确空状态，而不是退化为无反馈空白

## 次级动作

- `refresh` / `rescan` 属于正式动作，但在来源浏览工作区中应保持次级层级
- 来源浏览工作区可以提供进入来源根配置、刷新、重扫或修复流程的入口，但这些动作不应取代浏览主路径

## 与搜索工作区的关系

- 工作区切换器必须允许用户在 `Search` 与 `Inventory` 之间显式切换，而无需离开当前应用壳层
- 当前生产切片中，`Inventory` 在左侧栏中的可见标签可以表现为 `库管理`，用于强调它既承接来源浏览，也承接库级管理入口
- 搜索工作区可以保留来源摘要与进入来源浏览工作区的入口，但不应继续内嵌完整来源清单列表
- 从搜索工作区切换到来源浏览工作区，再切回搜索工作区时，查询草稿、结果列表与详情选中态必须保持
