# 100 文本搜索当前阶段计划

本计划承接 [spec.md](./spec.md) 的长期能力定义，收敛文本搜索当前阶段的首个可演示闭环。相关公开接口契约、错误载荷与资源引用对象编码统一复用 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md)，共享搜索工作区界面约束统一复用 [../008-ui-ux/search-workspace.md](../008-ui-ux/search-workspace.md)，本文不重复展开其详细 payload。详细测试设计见 [testing.md](./testing.md)。

## 概要

- 当前阶段交付形态固定为 `API + 最小 UI` 的可演示闭环，而不是纯后端原型
- 当前阶段来源范围固定为 `PDF + 图片`
- 当前阶段实际可返回的结果对象固定为 `document_page + image`
- 当前阶段索引主线固定为 `multivector first`
- 当前阶段首个实现模型固定为 `ColQwen3.5-4.5B-v3`
- 当前阶段检索后端固定为外部已运行的 `Qdrant` 实例

## 当前阶段工作流

### 1. 建库与选库

- 当前阶段用户必须先创建库或选择已有库，再进入文本搜索工作台
- 当前阶段的工作台必须始终显式展示当前目标库上下文
- 当前阶段创建库时，库配置必须显式启用 `multivector` 索引线；相关契约形状继续复用 `009`

### 2. 路径导入

- 当前阶段工作台必须提供路径选择 + 提交的本地导入入口，用于选择 PDF 或图片文件
- 当前阶段工作台可以提供仓库内置 fixture 快捷入口，用于快速验证真实索引与检索链路；该入口不替代正式路径导入契约
- 提交导入后，动作必须进入统一后台执行系统，而不是以旁路方式直接写成可搜索结果
- 批量导入采用“部分接受”语义：
  - 有效项进入同一个后台任务
  - 无效项通过逐项回执反馈原因
- 当前阶段只复用 `009` 中定义的导入动作契约，不在本文重复展开具体请求 / 响应字段

### 3. 嵌入式任务面板观察

- 当前阶段工作台内必须提供嵌入式任务面板，用于显示最近导入 / 索引任务的阶段、进度摘要与失败摘要
- 任务面板在当前阶段只承担观察职责，不提供取消、重试或恢复动作
- 当前阶段不实现独立任务中心，也不实现独立运行时健康页面
- 当前阶段任务观察面继续复用共享搜索工作区，不拆成独立页面

### 4. 文本搜索

- 当前阶段工作台只提供单文本框 + 执行按钮，不暴露高级过滤、调试或索引线选择控件
- 搜索始终作用于当前工作台所指向的单个目标库
- 当目标索引线尚未 ready 时，当前阶段必须向用户表达显式未就绪反馈，而不是只显示空结果
- 当前阶段文本查询继续复用共享搜索工作区；当前阶段只启用 text mode

### 5. 结果与详情

- 当前阶段结果列表默认以统一混排列表呈现 `document_page` 与 `image`
- 结果卡片只要求稳定呈现最小结果字段；字段契约继续复用 `004` 与 `009`
- 文档页详情在当前阶段展示前后页邻近上下文；图片详情展示同源基础信息

## 当前阶段约束

- 当前阶段不实现 `video_segment` 的实际索引、搜索结果与详情面板
- 当前阶段不实现 `single-vector` 可运行搜索线
- 当前阶段默认通过本地模型执行 `multivector` 文本搜索链路；后续阶段再扩展更多索引线与模型路线
- 当前阶段不实现 Qdrant 的本地进程托管、发现或启停管理；只假设已有外部实例可连通

## Deferred

- `video_segment` 的实际落地与可搜索结果
- 图片查询与视频查询
- 来源根扫描、规则管理、刷新与重扫
- 完整任务中心
- 独立运行时健康页
- Qdrant 托管
- `single-vector` 可运行实现
- 高级过滤、索引线选择与调试控件

## 阶段验收摘要

- 用户能够创建或选择启用了 `multivector` 的目标库，并在工作台中始终看到明确库上下文
- 用户能够通过路径选择入口提交 PDF / 图片导入批次，并在工作台中看到部分接受结果与嵌入式任务状态
- 在索引完成后，用户能够通过文本查询命中文档页与图片，并看到统一混排结果
- 用户能够在三栏工作台中连续浏览搜索结果，并通过右侧详情面板查看可视预览、文档页邻近上下文或图片同源基础信息
- 当索引未 ready 或导入 / 索引失败时，工作台向用户表达明确未就绪或失败反馈，而不是静默返回空结果

详细测试分层、场景矩阵、fixture 策略与 GPU smoke 设计见 [testing.md](./testing.md)。
