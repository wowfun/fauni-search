# 130 文档搜索当前阶段计划

本计划承接 [spec.md](./spec.md) 的长期能力定义，收敛文档搜索当前阶段的首个可演示闭环。相关公开接口契约、错误载荷、临时查询资产入口与资源引用对象编码统一复用 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md)，共享搜索工作区界面约束统一复用 [../008-ui-ux/search-workspace.md](../008-ui-ux/search-workspace.md)，本文不重复展开其详细 payload。详细测试设计见 [testing.md](./testing.md)。

## 概要

- 当前阶段交付形态固定为 `API + 最小 UI` 的可演示闭环，而不是纯后端原型
- 当前阶段文档搜索继续复用现有共享搜索工作区，而不是新开独立页面或独立工作区
- 当前阶段查询输入允许是整份文档，也允许是指定页范围的文档片段；未指定页范围时默认整份文档
- 当前阶段查询文档来源固定为：
  - 单个临时上传文档
  - 库内文档对象引用，且使用 `source_id`
- 当前阶段硬验收结果对象固定为 `document_page + image`；`video_segment` 保留为可选扩展命中，但不作为首轮验收阻塞项
- 当前阶段索引主线固定为 `multi_vector_late_interaction` VectorSpace 路线
- 当前阶段首个查询文档格式固定为 `PDF`

## 当前阶段工作流

### 1. 建库、选库与可搜索内容准备

- 当前阶段用户必须先创建库或选择已有库，再进入文档搜索工作台
- 当前阶段的工作台必须始终显式展示当前目标库上下文
- 当前阶段文档搜索继续复用正式导入与索引链路，先让 PDF、图片、视频等内容进入可搜索状态，再执行文档查询
- 当前阶段不为文档搜索单独新增跨库或旁路导入链路；库内容准备方式继续复用上游导入能力

### 2. Document Mode 与查询文档输入

- 当前阶段在共享搜索工作区中正式启用 `Document` 查询模式
- 当前阶段 `Document` 模式一次只允许一个查询文档输入，不实现多文档输入或文档 + 文本组合输入
- 当前阶段查询文档来源固定为：
  - 单个临时上传 PDF
  - 库内文档对象引用，且使用 `source_id`
- 当前阶段工作区主路径固定为“上传 PDF 作为查询文档”与“从结果 / 详情中复用库内 `document_page` 作为查询文档”；不单独提供库内 `source_id` 选择器
- 当前阶段查询页范围是可选输入：
  - 未指定时按整份文档查询
  - 指定时按 `start_page/end_page` 表示的文档片段查询
- 当前阶段若直接复用库内 `document_page` 结果作为新的查询文档，则内部固定转成 `source_id + 单页范围`
- 当前阶段页范围在 UI 上固定为数字输入控件；留空表示整份文档，只填起始页表示单页，同时填写起始页 / 结束页表示连续页范围；接口提交时统一使用 `start_page/end_page`
- 当前阶段不实现文档 URL 引用接入、拖拽上传、多区间页范围或文档 + 文本 refine

### 3. 文档搜索

- 当前阶段工作台在 `Document` 模式下必须通过正式文档搜索端点发起查询，而不是复用文本 / 图片 / 视频搜索端点或内部旁路
- 单次文档搜索请求只承接一个查询文档输入，并允许带一个可选页范围
- 搜索始终作用于当前工作台所指向的单个目标库
- 当前阶段文档搜索的核心任务固定为“用文档找相似页面 / 图像 / 时刻”，而不是先按整份文档聚合后再解释

### 4. 结果与详情

- 当前阶段结果列表默认以统一混排列表呈现 `document_page`、`image`，并允许在实现已具备时额外返回 `video_segment`
- 当前阶段结果按实际分数统一排序，不做人为按对象类型分桶
- `document_page` 结果详情继续复用现有详情与 PDF 预览链路
- `image` 与 `video_segment` 结果详情继续复用现有详情与预览链路

## 当前阶段约束

- 当前阶段只支持 `PDF` 作为查询文档格式，不实现 docx / pptx / xlsx
- 当前阶段页范围必须是单个连续区间；不实现多区间组合查询
- 当前阶段页范围 UI 只做数字输入，不实现 PDF 内可视页选、缩略图页选或拖拽式页范围选择
- 当前阶段不实现文档 + 文本组合查询
- 当前阶段不实现文档查询输入的长期保存、收藏或管理流
- 当前阶段库内对象引用只使用 `source_id` 作为正式公开编码；`document_page` 复用只是工作区内的派生便利路径
- 当前阶段不单独提供库内文档源浏览器或 `source_id` 选择器；库内对象复用以 `document_page` 结果派生为主

## Deferred

- docx / pptx / xlsx 查询文档支持
- 多区间页范围查询
- 文档 + 文本组合查询
- 文档 URL 引用接入
- 查询文档的长期保存、历史管理与复用
- 自动文件监听
- 来源根扫描、规则管理、刷新与重扫
- 完整任务中心
- 独立运行时健康页
- Qdrant 托管
- `single-vector` 可运行实现
- 高级过滤、索引线选择与调试控件

## 阶段验收摘要

- 用户能够创建或选择启用了 `multi_vector_late_interaction` content types 的目标库，并在工作台中始终看到明确库上下文
- 用户能够在共享搜索工作区切换到 `Document` 模式，并通过临时上传 PDF 提供查询文档
- 用户能够在 `Document` 模式下不指定页范围直接按整份文档发起搜索
- 用户能够在 `Document` 模式下指定页范围，以该文档片段作为实际查询输入发起搜索
- 用户能够在共享搜索工作区中把库内 `document_page` 结果对象直接作为新的查询文档发起搜索
- 在目标库内容已进入可搜索状态后，用户能够通过文档查询命中 `document_page`
- 在目标库内容已进入可搜索状态后，用户能够在统一结果列表中看到 `document_page + image` 的按分数混排结果
- 若当前阶段实现与 fixture 已支持跨文档到视频时刻的稳定召回，`video_segment` 命中可作为扩展加分项，但不作为首轮验收阻塞项
- 当查询文档无效、页范围非法、目标索引未 ready 或文档搜索失败时，工作台向用户表达明确失败或未就绪反馈，而不是静默返回空结果

详细测试分层、场景矩阵、fixture 策略与 GPU smoke 设计见 [testing.md](./testing.md)。
