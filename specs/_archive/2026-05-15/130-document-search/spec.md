# 130 文档搜索 (Document Search)

定义 FauniSearch 的文档搜索能力域，明确文档查询如何在单个库上下文中作用于已进入可搜索状态的内容，并返回可消费的 Asset 搜索结果。

## 关键术语 (Terminology)

- 文档搜索（Document Search）
- 查询文档（Query Document）
- 查询页范围（Query Page Range）
- 文档片段查询（Document Slice Query）

## 范围

- 单个库上下文中的文档查询能力
- 文档搜索的长期结果对象边界
- 查询文档在整份输入与指定页范围输入之间的长期语义
- 文档搜索与上游临时查询资产、摄取、索引、搜索接口契约之间的复用边界

范围外：
- 具体请求 / 响应字段、错误载荷、游标编码与资源引用对象形状
- 具体上传入口、临时资产生命周期编码与对象引用编码细节
- 具体 PDF 渲染、页聚合、模型、索引算法与检索后端实现
- 页范围控件、PDF 预览实现与具体工作区布局细节
- 当前阶段实现范围、阶段验收步骤与 UI 形态

## 设计原则

- 单库上下文（Single-Library Scope）：文档搜索始终作用于单个目标库，不发生跨库隐式搜索
- 单次单输入（Single Public Query Input）：单次公开文档搜索请求只承接一种文档查询输入，不伪装为文档 + 文本组合查询
- 文档优先（Document-First Query）：文档搜索由显式提供的查询文档驱动，而不是从结果列表中隐式拼装查询输入
- 范围显式（Explicit Range Semantics）：查询页范围是文档查询输入的一部分；未指定时默认为整份文档，指定时必须按该范围解释
- 显式输入状态（Explicit Query Input State）：无效、缺失、过期、不可读取或页范围非法的查询文档必须产生显式失败，而不是静默降级
- 事实源复用（Fact-Source Reuse）：文档搜索复用上游临时查询资产、摄取、索引、搜索与接口契约专题，不在本专题重复定义其稳定事实

## 能力边界

- 文档搜索只接受文档查询输入
- 文档搜索始终作用于单个目标库
- 文档搜索长期上允许以下结果对象参与命中：
  - `document_page`
  - `image`
  - `video_segment`
- 文档搜索只能依赖目标库内已启用且已进入可搜索状态的索引线
- 文档搜索结果的对象详情、邻近上下文与展开路径继续复用上游搜索与接口契约专题，不在本专题单独定义其公开形状

## 查询文档与页范围状态

- 文档搜索只有在当前公开查询输入已经进入可读取、可消费状态后，才应开始执行
- 查询文档的具体来源形态继续复用上游状态模型与接口契约专题；本专题只要求系统能够区分“当前查询输入可用”与“当前查询输入不可用”
- 查询页范围在公开语义上属于查询文档输入的一部分：
  - 当页范围缺失时，系统必须将查询解释为“整份文档”
  - 当页范围存在时，系统必须将查询解释为“指定文档片段”
- 查询页范围非法、越界、不可解析或与当前查询文档不匹配时，系统必须返回显式失败，而不是隐式回退到整份文档
- 通过临时查询资产进入系统的查询文档，在过期后必须从可用查询输入集合中被主动移除；过期后继续访问不得隐式续期
- 查询文档进入可用状态并不自动意味着目标库已经可被文档搜索；目标索引线的可搜索状态仍然独立判定

## 功能需求

### 查询

- `FR-1` 系统必须允许用户在单个目标库上下文中发起文档查询。
- `FR-2` 文档查询只能作用于目标库内已启用且已进入可搜索状态的索引线，不得发生跨库隐式搜索。
- `FR-3` 单次公开文档搜索请求只能承接一种文档查询输入，不得把文档与文本伪装成同一次公开搜索请求。
- `FR-4` 文档查询必须同时支持“整份文档”与“指定页范围文档片段”两种长期语义；未指定页范围时默认整份文档。
- `FR-5` 当查询文档不可用、页范围非法、超出支持范围或命中未启用 / 未就绪索引线时，系统必须返回显式拒绝、显式失败或显式未就绪语义，而不是静默降级、忽略或返回空结果。

### 结果

- `FR-6` 文档搜索长期上必须允许 `document_page`、`image` 与 `video_segment` 作为可命中的结果对象。
- `FR-7` 文档搜索结果必须能够按统一结果集返回，并支持根据实际分数进行混排，而不是要求按对象类型分桶返回。
- `FR-8` 文档搜索结果必须能够以稳定、可消费的最小结果卡片形式被展示，并支持后续通过上游搜索 / 详情能力继续展开。

### 查询输入与可搜索状态

- `FR-9` 系统必须在文档查询执行前验证当前查询文档已经处于可用状态，而不是在查询后把输入错误伪装为无结果。
- `FR-10` 内容只有在正式接入与相关索引完成后，才能进入可被文档搜索命中的状态。
- `FR-11` 当目标索引线尚未就绪或正式接入 / 索引 / 激活失败导致文档搜索不可用时，系统必须向用户表达明确未就绪或明确失败反馈，而不是静默返回空结果。

## 验收标准

- `AC-1` 当目标库中的相关内容已进入可搜索状态后，文档查询能够命中 `document_page` 结果。
- `AC-2` 当目标库中的相关内容已进入可搜索状态后，文档查询能够命中 `image` 结果。
- `AC-3` 当目标库中的相关内容已进入可搜索状态后，文档查询能够命中 `video_segment` 结果。
- `AC-4` 当查询同时命中多类 Asset 结果时，同一结果集中允许按实际分数混排返回这些结果。
- `AC-5` 当查询文档缺失、过期、损坏、不可读取或页范围非法时，系统返回明确失败，而不是静默返回空结果。
- `AC-6` 当请求涉及已启用但尚未就绪的索引线时，系统返回明确未就绪反馈，而不是静默返回空结果。
- `AC-7` 文档查询只作用于单个目标库，并且单次请求只承接一种文档查询输入，不发生跨库隐式搜索或隐式组合查询。
