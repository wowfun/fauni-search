# 120 视频搜索 (Video Search)

定义 FauniSearch 的视频搜索能力域，明确视频查询如何在单个库上下文中作用于已进入可搜索状态的视觉内容，并返回可消费的搜索结果。

## 关键术语 (Terminology)

- 视频搜索（Video Search）
- 查询视频（Query Video）
- 查询时间范围（Query Time Range）
- 视频片段结果（Video Segment Result）
- 时刻检索（Moment Retrieval）

## 范围

- 单个库上下文中的视频查询能力
- 视频搜索的长期结果对象边界
- 查询视频在整段输入与指定时间范围输入之间的长期语义
- 视频搜索与上游临时查询资产、摄取、索引、搜索接口契约之间的复用边界

范围外：
- 具体请求 / 响应字段、错误载荷、游标编码与资源引用对象形状
- 具体上传入口、临时资产生命周期编码与对象引用编码细节
- 具体视频切段算法、抽帧策略、模型、索引算法与检索后端实现
- 时间轴组件、播放器控件与视频预览实现细节
- 当前阶段实现范围、阶段验收步骤与 UI 形态

## 设计原则

- 单库上下文（Single-Library Scope）：视频搜索始终作用于单个目标库，不发生跨库隐式搜索
- 单次单输入（Single Public Query Input）：单次公开视频搜索请求只承接一种视频查询输入，不伪装为视频 + 文本组合查询
- 时刻优先（Moments First）：视频搜索的核心任务是根据整段视频或指定时间范围寻找相似时刻，而不是先按整段视频聚合后再解释
- 范围显式（Explicit Range Semantics）：查询时间范围是视频查询输入的一部分；未指定时默认为整段视频，指定时必须按该范围解释
- 显式输入状态（Explicit Query Input State）：无效、缺失、过期、不可读取或范围非法的查询视频必须产生显式失败，而不是静默降级
- 事实源复用（Fact-Source Reuse）：视频搜索复用上游临时查询资产、摄取、索引、搜索与接口契约专题，不在本专题重复定义其稳定事实

## 能力边界

- 视频搜索只接受视频查询输入
- 视频搜索始终作用于单个目标库
- 视频搜索长期上允许以下结果对象参与命中：
  - `video_segment`
  - `image`
  - `document_page`
- 视频搜索只能依赖目标库内已启用且已进入可搜索状态的索引线
- 视频搜索结果的对象详情、邻近上下文与展开路径继续复用上游搜索与接口契约专题，不在本专题单独定义其公开形状

## 查询视频与时间范围状态

- 视频搜索只有在当前公开查询输入已经进入可读取、可消费状态后，才应开始执行
- 查询视频的具体来源形态继续复用上游状态模型与接口契约专题；本专题只要求系统能够区分“当前查询输入可用”与“当前查询输入不可用”
- 查询时间范围在公开语义上属于查询视频输入的一部分：
  - 当时间范围缺失时，系统必须将查询解释为“整段视频”
  - 当时间范围存在时，系统必须将查询解释为“指定视频片段”
- 查询时间范围非法、越界、不可解析或与当前查询视频不匹配时，系统必须返回显式失败，而不是隐式回退到整段视频
- 通过临时查询资产进入系统的查询视频，在过期后必须从可用查询输入集合中被主动移除；过期后继续访问不得隐式续期
- 查询视频进入可用状态并不自动意味着目标库已经可被视频搜索；目标索引线的可搜索状态仍然独立判定

## 功能需求

### 查询

- `FR-1` 系统必须允许用户在单个目标库上下文中发起视频查询。
- `FR-2` 视频查询只能作用于目标库内已启用且已进入可搜索状态的索引线，不得发生跨库隐式搜索。
- `FR-3` 单次公开视频搜索请求只能承接一种视频查询输入，不得把视频与文本伪装成同一次公开搜索请求。
- `FR-4` 视频查询必须同时支持“整段视频”与“指定时间范围视频片段”两种长期语义；未指定时间范围时默认整段视频。
- `FR-5` 当查询视频不可用、时间范围非法、超出支持范围或命中未启用 / 未就绪索引线时，系统必须返回显式拒绝、显式失败或显式未就绪语义，而不是静默降级、忽略或返回空结果。

### 结果

- `FR-6` 视频搜索长期上必须允许 `video_segment`、`image` 与 `document_page` 作为可命中的结果对象。
- `FR-7` 视频搜索结果必须能够按统一结果集返回，并支持根据实际分数进行混排，而不是要求按对象类型分桶返回。
- `FR-8` 视频搜索结果必须能够以稳定、可消费的最小结果卡片形式被展示，并支持后续通过上游搜索 / 详情能力继续展开。

### 查询输入与可搜索状态

- `FR-9` 系统必须在视频查询执行前验证当前查询视频已经处于可用状态，而不是在查询后把输入错误伪装为无结果。
- `FR-10` 内容只有在正式接入与相关索引完成后，才能进入可被视频搜索命中的状态。
- `FR-11` 当目标索引线尚未就绪或正式接入 / 索引 / 激活失败导致视频搜索不可用时，系统必须向用户表达明确未就绪或明确失败反馈，而不是静默返回空结果。

## 验收标准

- `AC-1` 当目标库中的相关内容已进入可搜索状态后，视频查询能够命中 `video_segment` 结果。
- `AC-2` 当目标库中的相关内容已进入可搜索状态后，视频查询能够命中 `image` 结果。
- `AC-3` 当目标库中的相关内容已进入可搜索状态后，视频查询能够命中 `document_page` 结果。
- `AC-4` 当查询同时命中多类视觉对象时，同一结果集中允许按实际分数混排返回这些结果。
- `AC-5` 当查询视频缺失、过期、损坏、不可读取或范围非法时，系统返回明确失败，而不是静默返回空结果。
- `AC-6` 当请求涉及已启用但尚未就绪的索引线时，系统返回明确未就绪反馈，而不是静默返回空结果。
- `AC-7` 视频查询只作用于单个目标库，并且单次请求只承接一种视频查询输入，不发生跨库隐式搜索或隐式组合查询。
