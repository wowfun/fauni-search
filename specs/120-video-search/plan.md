# 120 视频搜索当前阶段计划

本计划承接 [spec.md](./spec.md) 的长期能力定义，收敛视频搜索当前阶段的首个可演示闭环。相关公开接口契约、错误载荷、临时查询资产入口与资源引用对象编码统一复用 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md)，共享搜索工作区界面约束统一复用 [../008-ui-ux/search-workspace.md](../008-ui-ux/search-workspace.md)，本文不重复展开其详细 payload。详细测试设计见 [testing.md](./testing.md)。

## 概要

- 当前阶段交付形态固定为 `API + 最小 UI` 的可演示闭环，而不是纯后端原型
- 当前阶段视频搜索继续复用现有共享搜索工作区，而不是新开独立页面或独立工作区
- 当前阶段查询输入允许是整段视频，也允许是指定时间范围的视频片段；未指定时间范围时默认整段视频
- 当前阶段查询视频来源固定为：
  - 单个临时上传视频
  - 库内视频对象引用，包括 `source_id` 与库内 `video_segment`
- 当前阶段实际可返回的结果对象固定为 `video_segment + image + document_page`
- 当前阶段索引主线固定为 `multi_vector_late_interaction` `vector_space` 路线
- 当前阶段首个实现模型固定为本地视觉模型链路；具体模型选择在实现专题中落地，不在本计划固定
- 当前阶段检索后端固定为外部已运行的 `Qdrant` 实例

## 当前阶段工作流

### 1. 建库、选库与可搜索内容准备

- 当前阶段用户必须先创建库或选择已有库，再进入视频搜索工作台
- 当前阶段的工作台必须始终显式展示当前目标库上下文
- 当前阶段视频搜索继续复用正式导入与索引链路，先让视频、图片、PDF 等内容进入可搜索状态，再执行视频查询
- 当前阶段不为视频搜索单独新增跨库或旁路导入链路；库内容准备方式继续复用上游导入能力

### 2. Video Mode 与查询视频输入

- 当前阶段在共享搜索工作区中正式启用 `Video` 查询模式
- 当前阶段 `Video` 模式一次只允许一个查询视频输入，不实现多视频输入或视频 + 文本组合输入
- 当前阶段查询视频来源固定为：
  - 单个临时上传视频
  - 库内视频对象引用，且支持：
    - `source_id`
    - 库内 `video_segment`
- 当前阶段查询时间范围是可选输入：
  - 未指定时按整段视频查询
  - 指定时按 `start_ms/end_ms` 表示的视频片段查询
- 当前阶段若直接复用库内 `video_segment` 作为查询输入，则默认使用该对象自身的时间范围，不再要求用户重新指定范围
- 当前阶段时间范围在 UI 上采用时间轴拖选，但接口提交时统一使用 `ms`
- 当前阶段不实现视频 URL 引用接入、拖拽上传、关键帧手选、多段范围组合或视频 + 文本 refine

### 3. 视频搜索

- 当前阶段工作台在 `Video` 模式下必须通过正式视频搜索端点发起查询，而不是复用文本 / 图片搜索端点或内部旁路
- 单次视频搜索请求只承接一个查询视频输入，并允许带一个可选时间范围
- 搜索始终作用于当前工作台所指向的单个目标库
- 当前阶段视频搜索的核心任务固定为“找相似时刻（moment retrieval）”，而不是视频内导航或找原视频文件

### 4. 结果与详情

- 当前阶段结果列表默认以统一混排列表呈现 `video_segment`、`image` 与 `document_page`
- 当前阶段结果按实际分数统一排序，不做人为按对象类型分桶
- `video_segment` 结果详情在当前阶段通过原视频的限时播放或等价预览表达对应时间范围，不要求先物理导出 clip 文件
- `image` 与 `document_page` 结果详情继续复用现有详情与预览链路

## 当前阶段约束

- 当前阶段只支持本地视频路径导入与本地查询视频上传，不实现视频 URL 引用接入
- 当前阶段时间范围必须是单个连续区间；不实现多区间组合查询
- 当前阶段不实现视频 + 文本组合查询
- 当前阶段不实现视频查询输入的长期保存、收藏或管理流
- 当前阶段不要求将每个 `video_segment` 物理导出为独立 clip 文件
- 当前阶段默认通过本地视觉模型执行 `multi_vector_late_interaction` 视频搜索链路；后续阶段再扩展更多索引线、查询来源与组合交互

## Deferred

- 视频 URL 引用接入
- 视频 + 文本组合查询
- 多区间时间范围查询
- 关键帧手选、拖拽上传、区域搜索
- 查询视频的长期保存、历史管理与复用
- 自动文件监听
- 来源根扫描、规则管理、刷新与重扫
- 完整任务中心
- 独立运行时健康页
- Qdrant 托管
- `single-vector` 可运行实现
- 高级过滤、索引线选择与调试控件

## 阶段验收摘要

- 用户能够创建或选择启用了 `multi_vector_late_interaction` content types 的目标库，并在工作台中始终看到明确库上下文
- 用户能够在共享搜索工作区切换到 `Video` 模式，并通过临时上传视频或库内视频对象引用提供查询视频
- 用户能够在共享搜索工作区中把库内 `video_segment` 结果对象直接作为新的查询视频片段发起搜索
- 用户能够在 `Video` 模式下不指定时间范围直接按整段视频发起搜索
- 用户能够在 `Video` 模式下指定时间范围，以该视频片段作为实际查询输入发起搜索
- 在目标库内容已进入可搜索状态后，用户能够通过视频查询命中 `video_segment`
- 在目标库内容已进入可搜索状态后，用户能够在统一结果列表中看到 `video_segment + image + document_page` 的混排结果
- 用户能够在同一工作区中展开 `video_segment` 详情，并看到对应时间范围的预览
- 当查询视频无效、时间范围非法、目标索引未 ready 或视频搜索失败时，工作台向用户表达明确失败或未就绪反馈，而不是静默返回空结果

详细测试分层、场景矩阵、fixture 策略与 GPU smoke 设计见 [testing.md](./testing.md)。
