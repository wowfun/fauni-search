# 110 图片搜索 (Image Search)

定义 FauniSearch 的图片搜索能力域，明确图片查询如何在单个库上下文中作用于已进入可搜索状态的视觉内容，并返回可消费的搜索结果。

## 关键术语 (Terminology)

- 图片搜索（Image Search）
- 查询图片（Query Image）
- 图片搜索结果（Image Search Result）
- 查询输入状态（Query Input State）

## 范围

- 单个库上下文中的图片查询能力
- 图片搜索的长期结果对象边界
- 查询图片进入可用状态后的搜索可用性、未就绪与失败反馈语义
- 图片搜索与上游临时查询资产、摄取、索引、搜索接口契约之间的复用边界

范围外：
- 具体请求 / 响应字段、错误载荷、游标编码与资源引用对象形状
- 具体上传入口、临时资产生命周期编码与对象引用编码细节
- 具体模型、索引算法、融合排序与检索后端实现
- 对象详情、邻近上下文与展开接口的具体公开编码
- 当前阶段实现范围、阶段验收步骤与 UI 形态

## 设计原则

- 单库上下文（Single-Library Scope）：图片搜索始终作用于单个目标库，不发生跨库隐式搜索
- 单次单输入（Single Public Query Input）：单次公开图片搜索请求只承接一种图片查询输入，不伪装为图片 + 文本组合查询
- 查询图片优先（Query Image First）：图片搜索由显式提供的查询图片驱动，而不是从结果对象中隐式派生查询输入
- 视觉对象优先（Visual Unit First）：图片搜索默认返回视觉单元结果，而不是先按源内容聚合
- 显式输入状态（Explicit Query Input State）：无效、缺失、过期或不可读取的查询图片必须产生显式失败，而不是静默降级
- 事实源复用（Fact-Source Reuse）：图片搜索复用上游临时查询资产、摄取、索引、搜索与接口契约专题，不在本专题重复定义其稳定事实

## 能力边界

- 图片搜索只接受图片查询输入
- 图片搜索始终作用于单个目标库
- 图片搜索长期上允许以下结果对象参与命中：
  - `document_page`
  - `image`
  - `video_segment`
- 图片搜索只能依赖目标库内已启用且已进入可搜索状态的索引线
- 图片搜索结果的对象详情、邻近上下文与展开路径继续复用上游搜索与接口契约专题，不在本专题单独定义其公开形状

## 查询输入状态

- 图片搜索只有在当前公开查询输入已经进入可读取、可消费状态后，才应开始执行
- 查询图片的具体来源形态继续复用上游状态模型与接口契约专题；本专题只要求系统能够区分“当前查询输入可用”与“当前查询输入不可用”
- 通过临时查询资产进入系统的查询图片，在过期后必须从可用查询输入集合中被主动移除；过期后继续访问不得隐式续期
- 查询图片缺失、过期、损坏、不可读取或超出支持范围时，系统必须返回显式失败，而不是静默返回空结果
- 查询图片进入可用状态并不自动意味着目标库已经可被图片搜索；目标索引线的可搜索状态仍然独立判定

## 功能需求

### 查询

- `FR-1` 系统必须允许用户在单个目标库上下文中发起图片查询。
- `FR-2` 图片查询只能作用于目标库内已启用且已进入可搜索状态的索引线，不得发生跨库隐式搜索。
- `FR-3` 单次公开图片搜索请求只能承接一种图片查询输入，不得把图片与文本伪装成同一次公开搜索请求。
- `FR-4` 当查询图片不可用、超出支持范围或命中未启用 / 未就绪索引线时，系统必须返回显式拒绝、显式失败或显式未就绪语义，而不是静默降级、忽略或返回空结果。

### 结果

- `FR-5` 图片搜索必须默认返回视觉单元粒度的结果，而不是先按源内容聚合后返回。
- `FR-6` 图片搜索长期上必须允许 `document_page`、`image` 与 `video_segment` 作为可命中的结果对象。
- `FR-7` 图片搜索结果必须能够以稳定、可消费的最小结果卡片形式被展示，并支持后续通过上游搜索 / 详情能力继续展开。

### 查询输入与可搜索状态

- `FR-8` 系统必须在图片查询执行前验证当前查询图片已经处于可用状态，而不是在查询后把输入错误伪装为无结果。
- `FR-9` 内容只有在正式接入与相关索引完成后，才能进入可被图片搜索命中的状态。
- `FR-10` 当目标索引线尚未就绪或正式接入 / 索引 / 激活失败导致图片搜索不可用时，系统必须向用户表达明确未就绪或明确失败反馈，而不是静默返回空结果。

## 验收标准

- `AC-1` 当目标库中的相关内容已进入可搜索状态后，图片查询能够命中 `document_page` 结果。
- `AC-2` 当目标库中的相关内容已进入可搜索状态后，图片查询能够命中 `image` 结果。
- `AC-3` 当目标库中的相关内容已进入可搜索状态后，图片查询能够命中 `video_segment` 结果。
- `AC-4` 当查询同时命中多类视觉对象时，同一结果集中允许混排返回这些结果。
- `AC-5` 当查询图片缺失、过期、损坏或不可读取时，系统返回明确失败，而不是静默返回空结果。
- `AC-6` 当请求涉及已启用但尚未就绪的索引线时，系统返回明确未就绪反馈，而不是静默返回空结果。
- `AC-7` 图片查询只作用于单个目标库，并且单次请求只承接一种图片查询输入，不发生跨库隐式搜索或隐式组合查询。
