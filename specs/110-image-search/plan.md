# 110 图片搜索当前阶段计划

本计划承接 [spec.md](./spec.md) 的长期能力定义，收敛图片搜索当前阶段的首个可演示闭环。相关公开接口契约、错误载荷、临时查询资产入口与资源引用对象编码统一复用 [009-interfaces-and-protocol-contracts](../009-interfaces-and-protocol-contracts/spec.md)，共享搜索工作区界面约束统一复用 [../008-ui-ux/search-workspace.md](../008-ui-ux/search-workspace.md)，本文不重复展开其详细 payload。详细测试设计见 [testing.md](./testing.md)。

## 概要

- 当前阶段交付形态固定为 `API + 最小 UI` 的可演示闭环，而不是纯后端原型
- 当前阶段图片搜索继续复用现有共享搜索工作区，而不是新开独立页面或独立工作区
- 当前阶段查询输入固定为单张查询图片；其来源允许是单张临时上传图片，或库内 `image` / `document_page` 对象引用
- 当前阶段实际可返回的结果对象固定为 `document_page + image`
- 当前阶段索引主线固定为 `multivector first`
- 当前阶段首个实现模型固定为 `ColQwen3.5-4.5B-v3`
- 当前阶段检索后端固定为外部已运行的 `Qdrant` 实例

## 当前阶段工作流

### 1. 建库、选库与可搜索内容准备

- 当前阶段用户必须先创建库或选择已有库，再进入图片搜索工作台
- 当前阶段的工作台必须始终显式展示当前目标库上下文
- 当前阶段图片搜索继续复用 `100-text-search` 已建立的正式导入与索引链路，先让 PDF / 图片内容进入可搜索状态，再执行图片查询
- 当前阶段不为图片搜索单独新增导入链路；库内容准备方式继续复用 `100` 当前阶段能力

### 2. Image Mode 与查询图片输入

- 当前阶段共享搜索工作区必须提供 `Image` 查询模式
- 当前阶段 `Image` 模式只允许用户一次提供一张查询图片；当前 UI 可通过文件选择或剪贴板粘贴提供该图片
- 查询图片上传后应进入临时查询资产链路，并获得稳定的临时引用供后续图片搜索请求使用
- 当前阶段临时查询图片必须具备过期与主动回收行为；过期资产及其预览文件不应在运行期无限堆积
- 当前阶段同时启用两种查询图片来源：
  - 单张临时上传图片
  - 库内 `image` / `document_page` 对象引用
- 当前阶段库内对象引用只启用 `image` 与 `document_page` 视觉单元；不启用其他对象类型直接作为查询图片
- 当前阶段不实现图片拖拽上传、圈选 / 裁剪、区域搜索或图片 + 文本组合输入
- 当前阶段不实现多图片粘贴、多图片上传或混合输入

### 3. 图片搜索

- 当前阶段工作台在 `Image` 模式下必须通过正式图片搜索端点发起查询，而不是复用文本搜索端点或内部旁路
- 单次图片搜索请求只承接一张查询图片，不承接额外文本 refine 输入
- 搜索始终作用于当前工作台所指向的单个目标库
- 当查询图片不可用、目标索引线尚未 ready 或搜索失败时，当前阶段必须向用户表达显式失败或显式未就绪反馈，而不是只显示空结果

### 4. 结果与详情

- 当前阶段结果列表默认以统一混排列表呈现 `document_page` 与 `image`
- 当前阶段结果、详情、预览与任务观察面继续复用共享搜索工作区，不新开图片搜索专属详情页
- 结果卡片只要求稳定呈现最小结果字段；字段契约继续复用 `004` 与 `009`
- 文档页详情在当前阶段展示前后页邻近上下文；图片详情展示同源基础信息

## 当前阶段约束

- 当前阶段不实现图片 + 文本组合查询
- 当前阶段不实现除 `image` / `document_page` 之外的库内对象直接作为查询图片
- 当前阶段不实现圈选 / 裁剪 / 区域搜索
- 当前阶段不实现 `video_segment` 的实际命中、详情与预览
- 当前阶段不实现查询图片的长期保存、收藏或管理流
- 当前阶段默认通过本地模型执行 `multivector` 图片搜索链路；后续阶段再扩展更多索引线、查询来源与组合交互
- 当前阶段临时查询图片的回收由 Rust 主服务运行期维护逻辑承接，不暴露独立用户控制面

## Deferred

- `video_segment` 的实际落地与图片搜索结果
- 图片 + 文本组合查询
- 除 `image` / `document_page` 之外的其他库内对象直接作为查询图片
- 查询图片圈选、裁剪与区域搜索
- 查询图片的长期保存、历史管理与复用
- 视频查询
- 来源根扫描、规则管理、刷新与重扫
- 完整任务中心
- 独立运行时健康页
- Qdrant 托管
- `single-vector` 可运行实现
- 高级过滤、索引线选择与调试控件

## 阶段验收摘要

- 用户能够创建或选择启用了 `multivector` 的目标库，并在工作台中始终看到明确库上下文
- 用户能够在共享搜索工作区切换到 `Image` 模式，并通过文件选择或剪贴板粘贴提供一张查询图片
- 用户能够在共享搜索工作区中把库内 `image` 结果对象直接作为查询图片再次发起图片搜索
- 用户能够在共享搜索工作区中把库内 `document_page` 结果对象直接作为查询图片再次发起图片搜索
- 在目标库内容已进入可搜索状态后，用户能够通过图片查询命中 `document_page` 与 `image`
- 用户能够在统一结果列表中浏览图片搜索结果，并通过同一工作区内的详情面板查看可视预览、文档页邻近上下文或图片同源基础信息
- 当查询图片无效、目标索引未 ready 或图片搜索失败时，工作台向用户表达明确失败或未就绪反馈，而不是静默返回空结果

详细测试分层、场景矩阵、fixture 策略与 GPU smoke 设计见 [testing.md](./testing.md)。
